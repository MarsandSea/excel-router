# 常见问题 FAQ

## 杀毒软件误报

**Q：下载的 exe 被 Windows Defender / 杀毒软件报「木马」「病毒」，能不能用？**

能用，这是**误报**（False Positive），不是真的有病毒。原因：

- 单文件版（`ExcelRouter-vX.X.X.exe`）每次启动会把内部的 Python 运行时和依赖
  解压到系统临时目录再执行，这种「自解压后执行」的行为模式和部分木马的启动
  方式相似，容易触发杀毒软件的**启发式扫描**误判。
- 本工具用 PyInstaller 打包，官方预编译的打包引导程序（Bootloader）也被
  许多其他打包了恶意程序的软件用过，导致该 Bootloader 的特征码被一些厂商
  标记为可疑，这与本项目代码本身无关。

**怎么办：**
1. **优先下载 ZIP 版**（`ExcelRouter-vX.X.X-win64.zip`，解压后运行里面的 exe）——
   这种打包方式不需要运行时自解压，基本不会触发上述误报。
2. 如果杀毒软件拦截了，选择「允许运行」/「信任此文件」，或临时关闭实时防护后运行一次。
3. 本项目完全开源，代码在 [GitHub 仓库](../) 可自行审查，也欢迎自己用
   `pip install -r requirements.txt && python main.py` 从源码直接运行，
   或者自己用 `build.bat` 本地打包，生成的文件不经过第三方分发，更不容易被拦截。
4. 如果你想帮忙推动误报修复，可以去
   [Microsoft 误报反馈门户](https://www.microsoft.com/en-us/wdsi/filesubmission)
   提交一次，通常几天内云端库会更新。

## 拆分后某一列（如「到账金额」）是空白的

**Q：数据源里那一列明明有公式和数字，拆分出来却是空白？**

本工具通过 pandas 读取的是 Excel **公式的缓存计算结果**，不会重新计算公式。

- 如果这份 Excel 是**用 Excel/WPS 打开并保存过**的，公式一定带缓存值，正常显示。
- 如果这份 Excel 是**程序生成、或公式从未在 Excel 里被计算/保存过**，就没有
  缓存值，读出来就是空白。

**怎么办：** 用 Excel 或 WPS 打开这份表格，`Ctrl+S` 直接另存（触发一次公式计算），
再用本工具处理。工具本身也会在处理前**自动检测**这种情况并在日志里提前预警，
不会让你在拆完之后才发现数据是空的。

## `.xls`（旧版 Excel 格式）支持到什么程度

`.xls` 文件会被自动转换成 `.xlsx` 再处理，**能拆分、数据完整**，但转换过程中
**无法保留原始的字体、颜色、边框等格式**，只保留数据本身。如果格式很重要，
建议先用 Excel/WPS 把 `.xls` 另存为 `.xlsx` 再处理。

---

## 数据区的合并单元格会保留吗

**不会。** 数据区域（表头以下的数据行）如果有合并单元格，拆分后会被展开为
普通单元格（每行独立填值）。**表头区域的合并单元格会正常保留**（比如大标题
跨列合并、分组标签合并等），这也是三层复杂表头能被正确识别的关键设计。

## 单文件版启动比较慢，正常吗

正常。单文件版（onefile）每次启动都要先把依赖解压到临时目录，这个过程通常
需要几秒钟，属于 PyInstaller onefile 模式的固有特性。如果比较介意，改用
ZIP（onedir）版——解压一次后，之后每次启动都是直接运行文件夹里的 exe，
没有这个解压过程，启动明显更快。

## 处理大文件（几万行）时界面卡住、像死机了一样，是不是崩溃了

**不是崩溃，是真的在算。** v2.4 之前，进度条只按「文件数」推进——如果输入只有 1 个大文件，
进度条一开始就跳到头，中间处理的十几秒到几十秒里界面看起来毫无反应，容易被误认为卡死。

v2.4 起已改进：
- 进度条现在反映**文件内部**的真实阶段（读取 → 读格式 → 逐组拆分 → 保存），几万行的大文件
  也会平滑推进，不会卡在某个数字不动。
- 点击「开始处理」的一瞬间，进度条立刻变成滚动动画、日志立刻打出「已开始，正在扫描输入…」，
  不会有几秒钟的静默期。
- 日志里会每写入约 2000 行打一条「…已拆分写入 N 行」的心跳，大文件也能看到进度在走。
- 处理过程中让出了 CPU 时间片给界面线程，窗口不会失去响应（可以正常最小化/拖动）。

如果升级后仍然长时间（几分钟以上）没有任何新日志、进度条也不动，才说明可能真的卡住了，
请提 [Issue](../../issues) 并附上大致的行数和列数。

## 「到人」+「ZIP」这套输出结构具体是怎样的

开启「到人」后，每次拆分会同时产出**两套**结果：

- **汇总**：按主拆分列（如「部门」）拆分、跨文件合并后的完整表
- **到人**：在汇总基础上，再按你指定的第二列（如「姓名」）拆到每个人的独立文件

批量模式（输入是文件夹）下，每个主取值会打包成一个 ZIP，比如 `销售部.zip`，
里面同时包含 `汇总/`（该部门的完整数据）和 `到人/`（该部门每个人的单独文件），
可以直接把整个 ZIP 发给对应部门负责人。
